## What This Case Is

A public-safe reasoning archive for Violin Stand Partner, a bounded continuity and judgment-support system for violinists and parents between lessons. The case records how a broad AI violin coach concept was narrowed into a stand partner: teacher authority remains intact, daily practice is reduced to one page turn, source provenance is separated from AI synthesis, durable memory requires human authority, and First Response becomes a protective layer rather than a broad repair claim.

The public archive demonstrates authorship, product restraint, and architectural judgment without publishing exact prompts, Approved Compacts, route trees, thresholds, memory schemas, source-ranking logic, platform workarounds, launch gates, pricing, or evaluation fixtures.

## The Founding Argument in One Sentence

Violin Stand Partner was not created by asking AI to design a violin app; it emerged from human judgments about where AI should help, where it should stop, what evidence it should preserve, and which product obligations had not earned the right to exist.

## Visual Timeline

1. Broad AI violin coach rejected in favor of bounded continuity.
2. Teacher replacement rejected; between-lessons support becomes the problem space.
3. Parents become first-class users, forcing actor-sensitive routing.
4. Comprehensive practice planning narrows to a five-minute page turn.
5. One active concern expands into multiple open pages, then contracts to one current action per page.
6. Coach is demoted from product surface to orchestration layer.
7. Freeform memory becomes visible, approved, useful state.
8. Conversational source retrieval becomes a reviewed segment-level corpus.
9. Safety disclaimers become behavioral boundaries.
10. First Response moves from repair help to protective, event-driven judgment support.

## Major Reversals

| Earlier direction | Pressure applied | Result |
|---|---|---|
| Broad AI violin coach | Authority and attribution risk | Bounded stand partner |
| Complete practice plan | Cognitive overload | One five-minute page turn |
| One active concern | Real practice is parallel | Multiple open pages |
| Unlimited parallel action | Platform state failure | One action per page |
| Dynamic regeneration | Instability and leakage | Stable daily action |
| Coach as whole product | Hidden state | Coach as orchestration layer |
| AI remembers everything | Probabilistic memory risk | Approved durable state |
| Freeform source search | Misattribution risk | Reviewed segment corpus |
| Disclaimers | Did not constrain behavior | Top-level behavioral boundaries |
| Repair feature | Wrong authority claim | Diagnostic First Response |
| Diagnose immediately | Parent stress and handling risk | Scene stabilization first |
| Full video for every route | Excessive launch work | Tiered media |
| First Response daily ritual | Weak recurrence and possible anxiety | Always-visible event utility |
| First Response first app | Weak daily retention and product obligation | Embedded in Stand Partner |
| First Response second standalone | Weaker ritual than Decision Tree | Decision Tree remains stronger second standalone app |

## Generalization Boundary

This case publishes a generalized reasoning chain, not the production implementation. Exact prompts, Approved Compacts, routing logic, authorization thresholds, source mappings, memory structures, evaluations, test fixtures, deployment details, and commercially sensitive sequencing are intentionally omitted. The public case documents why the system is shaped this way without publishing the complete machinery required to reproduce it.

## Cascade Rules

If the product begins presenting itself as a general violin coach or teacher replacement, revisit C12-001 through C12-006 before launch language is published.
If parent routes authorize the same actions as player routes without actor-sensitive review, revisit C12-008 through C12-011.
If the daily surface generates broad plans instead of one bounded action, revisit C12-012 through C12-019.
If Coach hides state, attribution, or uncertainty, revisit C12-020 through C12-024.
If the interface lets AI judgment outrank the player, parent, teacher, luthier, or qualified professional, revisit C12-025 through C12-029.
If durable memory can be written without visibility or correction, revisit C12-030 through C12-033.
If Ben-source retrieval cannot distinguish exact source, related source, synthesis, and no-match, revisit C12-034 through C12-037.
If First Response stretches a route to fit an uncertain case, revisit C12-038 through C12-042; no-match and non-intervention are valid outcomes.
