# C13-008: Use Reference Images to Eliminate Unnecessary Decisions

**Method:** YY Method™ Home Edition v2.3 — Capture → Why → Why-Not → Commit → Timestamp

**Status:** Decided

**Date:** 2026-07-24

**Domain:** Visual Locking

**Depends On:** C13-007

**Freshness Boundary:** Active. Revisit if future systems consistently preserve identity, wardrobe, props, and environments from text alone.

**Source:** Public-safe synthesis from iterative AI video production. Exact prompts, generated clips, reference libraries, and platform-specific production fixtures remain private.

---

## Capture

Text descriptions of people, props, and locations remained visually unstable.

Characters changed face, age, hairstyle, and clothing. The violin, case, room, and furniture drifted. A successful generation could not guarantee that the next shot belonged to the same world.

---

## Why

The words accurately described the intended nouns.

It seemed reasonable that repeating the same description would reproduce a sufficiently similar person or environment.

---

## Why-Not

A description identifies a category, not an exact appearance.

The generator still chose facial structure, proportions, wardrobe details, lighting, pose, object design, and room layout. Each choice reopened continuity.

The scar was realizing that repeated language does not preserve a visual decision. It merely repeats the request to make it again.

---

## Commit

**Decision:** Establish important identities, props, wardrobe, environments, and compositions with approved reference images.

When a satisfactory visual decision exists, preserve and reuse it rather than asking the model to recreate an equivalent from prose.

The image now carries many of the stable decisions. This creates a division of labor between image and prompt that must be made explicit.
